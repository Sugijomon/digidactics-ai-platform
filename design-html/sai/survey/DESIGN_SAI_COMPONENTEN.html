Lovable component: ScanNav.tsx

Props: onBack: () => void · onNext: () => void · nextDisabled: boolean · optioneel nextLabel?: string (default "Volgende"). De scheidingslijn en padding zitten IN de component — niet in het scherm zelf.

Implementatiestrategie
Lovable: drie componenten, één keer bouwen
Ja — dit zijn precies de componenten die je in Lovable als gedeelde React-componenten bouwt. Zelfde principe als een HTML-layout met vaste header en footer. Elk scherm importeert de drie componenten en vult alleen de inhoud van de glass card in.
// Elke stap-pagina ziet er zo uit:
<ScanHeader />
<ScanProgress current={2} total={5} />

<div className="glass-card ...">
  // Sectielabel + H2 + subtekst
  // Vragen / opties / content
  
  <ScanNav
    onBack={() => navigate(-1)}
    onNext={() => navigate('/stap-3')}
    nextDisabled={!antwoordGekozen}
  />
</div>
Voordeel
Logo-wissel, kleurwijziging of knoptekst later aanpassen? Één bestand, alle schermen bijgewerkt.
Lovable prompt
Geef dit spec-document + de HTML van één scherm mee. Zeg: "Extraheer ScanHeader, ScanProgress en ScanNav als gedeelde componenten."
Volgorde
Bouw eerst de drie componenten. Dan pas de schermen — anders bouw je ze inline en moet je later refactoren.
