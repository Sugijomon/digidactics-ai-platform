# Shadow AI Dashboard V2

## Doel

Deze V2 verwerkt de belangrijkste inhoudelijke gaten uit de eerste dashboardversie, met drie uitgangspunten:

- survey-inhoud is anoniem;
- disclosure geldt alleen voor ambassadeurs die zichzelf expliciet aanmelden;
- deelname-status mag operationeel gevolgd worden voor reminders, maar niet als naamgerichte rapportage worden gebruikt.

## Gap Matrix

| Onderwerp | In V1 | V2-aanpak |
|---|---|---|
| Gebruikfrequentie | gedeeltelijk | toegevoegd als organisatie-inzicht en adoptiepatroon |
| Non-use redenen | ontbreekt | toegevoegd als analyseblok voor niet-gebruikers |
| Scheiding governance-lagen | gedeeltelijk | explicieter gemaakt: catalogus, org policy, EU AI Act-signaal |
| Triagevelden `highest_risk_use_case` / `highest_risk_context` / `override_reason` | gedeeltelijk | explicieter gemaakt in overview en review-export |
| Context als aparte laag | gedeeltelijk | toegevoegd in review- en analyseblokken |
| Audit package | impliciet | expliciet rapporttemplate toegevoegd |
| Open review export | impliciet | expliciet rapporttemplate toegevoegd |
| Naamgerichte rapportage | aanwezig in mockups | verwijderd of omgezet naar anonieme case-/clusterlogica |
| Ambassadeurslogica | niet scherp genoeg | verduidelijkt als vrijwillige opt-in |
| Deelname-monitoring | niet scherp genoeg | verduidelijkt als operationele reminderlaag, niet als inhoudelijke rapportage |

## Aanpassingen Per Pagina

### 01. Activatie & status

- disclosure-sectie verduidelijkt naar opt-in ambassadeurschap en reminderbeheer;
- deelnemerslijst geherpositioneerd als operationele uitnodigingslaag;
- expliciete noot toegevoegd dat naamniveau niet in analyse- of managementrapportages komt.

### 02. Tool inventaris

- governance-lagen explicieter gemaakt;
- DPO-matrix aangescherpt als anonieme combinatie-analyse;
- naamniveau uit de matrixdetails verwijderd.

### 03. Overzicht

- gebruiksfrequentie toegevoegd;
- non-use redenen toegevoegd;
- governance-lagen zichtbaar gemaakt;
- triage verrijkt met context, use case en override-reden;
- caseweergave aangescherpt als anoniem.

### 04. Governance

- readiness/awareness en innovatie explicieter onderscheiden;
- contextsignalen toegevoegd;
- non-use als apart adoptiesignaal toegevoegd.

### 05. Reporting

- audit package toegevoegd als rapporttemplate;
- open reviews export toegevoegd als rapporttemplate;
- historie geneutraliseerd zodat geen persoonniveau nodig is.

### 06. Progress

- voortgang aangevuld met adoptie- en frequentie-inzichten;
- interpretatie op groepsniveau expliciet gehouden.
